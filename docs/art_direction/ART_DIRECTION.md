# 《记忆背包》美术方向重定稿

更新时间：2026-06-02

本文用于约束后续 image2 资产生成、Godot 接入和人工验收。后续不要只按单张素材好不好看判断，而要看整屏是否统一、是否服务玩法、是否符合剧本文档。

## 1. 核心气质

整体方向：颜色鲜艳的日系童话风。

关键词：
- 明亮、清爽、柔和、童话绘本感。
- Q 版角色可爱但不能幼稚廉价。
- 场景要有道路、地面和角色落脚点，人物不能漂浮。
- UI 要像轻巧的布袋、木牌、纸张、花草和缎带组合出来的游戏道具。
- 背包是核心玩法视觉中心，必须比普通状态栏更有存在感。

禁止项：
- 过暗。
- 过油腻。
- 欧美魔幻。
- AI 细节糊。
- 金属太多。
- UI 太厚重。
- 大面积棕黑皮革和暗木板。
- 地狱、恐怖、哥特、硬核盔甲、厚重石雕。
- 角色和 UI 风格互相割裂。

## 2. 色彩规则

推荐主色：
- 草绿、嫩黄绿、奶油白、天空蓝、樱花粉、珊瑚红、浅木色。

推荐点缀：
- 少量金色高光、花瓣、叶片、浅色布纹、纸张边框。

限制：
- 棕色只能作为木框或背包边缘的辅助色，不能统治画面。
- 蓝灰和暗紫只能用于敌人或阴影，且面积要小。
- 战斗画面也要保持明亮，敌人的威胁感来自造型和动作，不靠压暗整屏。

## 3. 材质规则

优先材质：
- 软布、亚麻、纸、彩绘木头、藤编、花草、缎带、陶瓷小饰物。

谨慎使用：
- 皮革、黄铜、深色木头。

尽量不用：
- 钢铁、铠甲、厚金属扣、大块宝石、暗黑魔法纹样。

## 4. 屏幕构图基准

探索/行走：
- 上半区是当前场景背景，上层叠加 Q 版主角行走表现。
- 主角团视觉重心居中偏下，脚要踩在道路或地面上。
- 下半区在非对话状态展示记忆背包操作 UI。
- 对话状态展示 Galgame 对话框，背包 UI 收起或弱化，避免互相遮挡。

战斗：
- 上半区左侧是 Q 版主角，右侧是 Q 版敌人。
- 双方必须站在同一地面透视上。
- 攻击演出要从角色动作、位移、受击反馈、斩击特效共同组成。
- 下半区仍然保留背包操作区，让玩家感到记忆道具随时能影响战斗。

背包 UI：
- 左侧是弃置/垃圾堆。
- 中央是 2x2 记忆背包格。
- 右侧是新发现记忆。
- 每个区域要像真实可触碰的袋子或小板子，不能只是代码画线。

## 5. 概念图候选

### 方案 1：探索与背包平衡

文件：`docs/art_direction/concepts/concept_01_travel_backpack.png`

优点：
- 颜色、光线和童话绘本感接近目标。
- 主角走在道路上，落地感较好。
- 下方背包结构清楚，左中右分区明确。

问题：
- 战斗张力不足。
- UI 偏纸艺和装饰，正式接入时需要减少花边，保留更清晰的可操作槽位。

适合采用：
- 探索场景、行走构图、背包基础分区。

### 方案 2：战斗商业化屏幕

文件：`docs/art_direction/concepts/concept_02_battle_backpack.png`

优点：
- 最接近“左主角、右敌人、下方背包”的完整游戏屏幕。
- 角色、敌人、地面和 UI 的整屏关系清楚。
- 敌人可爱但有压迫感，适合 Q 版战斗方向。

问题：
- 队伍人数偏多，可能偏离 MVP 主角表达。
- 神社/和风地域符号偏强，后续要回到剧本文档里的村庄、森林、记忆主题。

适合采用：
- 战斗站位、敌人尺度、角色落地关系、整屏层级。

### 方案 3：背包核心玩法界面

文件：`docs/art_direction/concepts/concept_03_memory_board.png`

优点：
- 视觉最鲜艳、最商业化。
- 背包 UI 轻巧、清楚、符合“记忆背包是核心”的诉求。
- 中央 2x2 格子有亲和力，适合让玩家拖动物品。

问题：
- 无关小伙伴过多，容易偏离严肃一点的剧本主题。
- 和风小镇元素过强，需要替换成遗忘森林和村庄主题。

适合采用：
- 背包 UI 材质、色彩饱和度、界面亲和力。

## 6. 推荐结论

第一轮概念图反馈：三张图过于同质化，整体偏浮夸，背包被错误理解为 2x2 小背包。新的方向必须把背包改成“多格可摆放棋盘”，并让不同方案真正拉开风格距离。

建议不再直接采用第一轮任一方案作为最终基准，只保留以下拆解结论：
- 方案 1 的道路透视可参考。
- 方案 2 的左主角右敌人站位可参考。
- 方案 3 的轻量布包材质可参考。

最终 UI 方向必须以第二轮多格背包概念为准。

落地时不要直接复刻任何一张图，而是拆成三条资产规则：
- UI 采用方案 3 的明亮布包、花草、软边框和大格子。
- 战斗采用方案 2 的左主角右敌人、同一地面、上半区演出。
- 探索采用方案 1 的道路透视、主角居中、远处事件点。

## 8. 第二轮多格背包候选

第二轮目标：参考“背包合成/格子摆放”类玩法的视觉逻辑，让记忆背包成为很多格子的核心操作棋盘，而不是 4 个装饰槽。

### 方案 A：清爽扁平动画 UI

文件：`docs/art_direction/concepts_v2/concept_v2_01_clean_grid.png`

优点：
- 多格背包、左右区域、不同尺寸物件都很清楚。
- 色彩清爽，商业手游感强，后续 Godot 接入难度较低。

问题：
- 偏休闲可爱，记忆主题的温柔失落感不足。

适合采用：
- 背包系统主结构、格子清晰度、拖拽玩法表达。

### 方案 B：克制绘本水粉

文件：`docs/art_direction/concepts_v2/concept_v2_02_gouache_grid.png`

优点：
- 物品占位最像“背包合成”玩法。
- 道具形状和格子关系非常明确。

问题：
- 色彩偏旧、偏朴素，不够鲜艳。
- 气质更像乡村桌游，商业游戏冲击力弱。

适合采用：
- 多尺寸记忆物件设计、背包布面格子结构。

### 方案 C：纸艺拼贴手账

文件：`docs/art_direction/concepts_v2/concept_v2_03_papercraft_grid.png`

优点：
- 风格差异最大，有独特辨识度。
- 纸张、缝线、贴片很适合“记忆被收藏”的主题。

问题：
- 色彩不够鲜艳。
- 动作和战斗表现可能偏静态。

适合采用：
- 记忆物件材质、背包格子触感、收藏感。

### 方案 D：明亮 JRPG 战斗背包

文件：`docs/art_direction/concepts_v2/concept_v2_04_tactical_grid.png`

优点：
- 最接近可直接落地的游戏战斗 UI。
- 上方战斗、下方多格背包的屏幕结构清楚。
- 格子数量、物品占位、操作区域最符合 MVP 系统。

问题：
- 部分物件偏普通装备和 JRPG 道具，后续要替换成剧情记忆物件。
- 需要减少金属扣和望远镜、晶石等与剧本无关的道具。

适合采用：
- 战斗屏幕结构、多格背包规格、Godot UI 落地基准。

第二轮推荐：
- 系统结构选方案 D。
- 色彩清爽度选方案 A。
- 记忆主题材质选方案 C。
- 物品占位规则参考方案 B。

下一轮正式资产不要再生成整张概念图，而应拆成可接入资产：
- 背包底板：多格棋盘，建议 7x4 或 8x3。
- 记忆物件图标：按 1x1、1x2、2x1、2x2、L 形分别设计。
- 弃置区和新发现区：左右侧袋。
- 战斗上半区：单独生成场景背景和 Q 版角色，不和背包底板烘在同一张图里。

## 9. 最终选型基准

最终概念图：`docs/art_direction/final/final_memory_backpack_art_direction.png`

用户确认的组合方案：
- 结构选方案 D：上半区战斗/行走，下半区大背包。
- 清爽度选方案 A：明亮、干净、可读，不再浮夸。
- 记忆材质借方案 C：纸、布、缝线、旧照片、叶片、手账感。
- 物品占格规则参考方案 B：物件按不同形状和尺寸占据背包格子。

最终基准图验收结论：
- 上半区角色和敌人左右站位清楚，并站在同一地面上。
- 下半区背包是多格棋盘，而不是 2x2 或 4 格装饰槽。
- 格子边界可读，空格可见，适合后续做拖拽玩法。
- 记忆物件包括汤碗、姓名缎带、木剑、照片、折叠地图、叶枝、小袋等，方向符合剧本主题。
- 色彩比第一轮克制，保留清爽日系童话感。

后续正式资产生成规则：
- 背包底板必须单独生成，透明或可裁切，不和背景/角色烘在一起。
- 背包格子建议先做 `7x4`，必要时在 Godot 中通过代码叠加真实交互格。
- 记忆物件必须按占格尺寸分批生成：`1x1`、`1x2`、`2x1`、`2x2`、`L` 形。
- 每个物件都必须对应剧本文档中的记忆，不再生成望远镜、晶石、普通装备等无关道具。
- 战斗场景、Q 版主角、Q 版敌人、背包 UI 分层接入，不能再把整张概念图直接当作游戏 UI。
- 任何新方案如果回到 4 格背包、厚重金属 UI、暗沉欧美魔幻、浮夸花饰，都判定为不通过。

## 10. 成熟化修订基准

成熟化概念图：`docs/art_direction/final/final_memory_backpack_art_direction_v2_mature.png`

修订原因：
- 第一张最终图结构正确，但画风偏低龄，角色和敌人有玩具/毛绒感。
- 游戏主题是记忆、失去、姓名和关系保留，不适合过度幼态、糖果色或儿童冒险气质。

成熟版新基准：
- 角色改为少年向 Q 版，不是幼童比例。
- 敌人改为无名纸片残影、空白面具、破碎记忆感，不做圆润毛绒怪。
- 色彩保持清亮，但降低糖果粉和过度饱和。
- 场景可以有柔和忧伤、遗忘森林和墓碑/路牌等失忆暗示，但不能变成暗黑恐怖。
- 背包仍保持多格占位玩法，不能因为成熟化而牺牲格子可读性。

新增禁止项：
- 低龄玩具感。
- 毛绒玩偶式敌人。
- 过度圆润的糖果色 UI。
- 儿童绘本嘉年华气质。
- 为了成熟而过度压暗、变灰、变写实。

当前优先级：
- 正式资产以成熟化概念图为气质基准。
- 背包结构继续沿用最终选型基准。
- 色彩要在成熟版基础上略微提高清亮度，但不能回到幼稚糖果感。

## 11. A/B/C 画风试验

当前成熟版定义为 A：
- 文件：`docs/art_direction/final/final_memory_backpack_art_direction_v2_mature.png`
- 定位：结构正确，主题贴合，低龄感已明显降低。
- 问题：仍有一定 AI 合成味，色彩偏灰绿，角色和背景的人工笔触不足。

### B：精致日系幻想 RPG 插画方向

文件：`docs/art_direction/style_trials/style_b_refined_jrpg.png`

定位：
- 更接近精致商业 RPG 插画。
- 角色线条、敌人设计和战斗张力比 A 更强。
- 适合参考角色、敌人、战斗演出和宣传视觉。

优点：
- 主角和敌人的气质更成熟。
- 细节密度高，商业感更强。
- 背包多格结构仍然保留。

风险：
- 画面偏暗，森林和 UI 的灰度较重。
- 容易滑向普通幻想 RPG，而不是“记忆背包”的独特主题。
- 后续如果采用，需要提高底部 UI 的清亮度，并减少过度精致带来的 AI 味。

### C：手绘动画电影自然童话方向

文件：`docs/art_direction/style_trials/style_c_handdrawn_film.png`

定位：
- 更接近手工绘制、自然、朴素的叙事动画感。
- 强调生活物件、旧纸、布料、自然光和失去感。
- 适合降低 AI 味，并贴近“记忆与关系”的主题。

优点：
- 三个方向里 AI 味最低。
- 人物比例更成熟，低龄玩具感少。
- 记忆背包像真实手工物件，主题表达更稳。

风险：
- 商业游戏冲击力略弱。
- 色彩偏土、偏旧，可能需要后期提高亮部和角色识别度。
- UI 边框和格子如果照搬，可能显得不够精致。

阶段建议：
- 如果优先“商业感”和角色吸引力，选 B 做基础，再把亮度和背包 UI 拉回 A。
- 如果优先“主题贴合”和降低 AI 味，选 C 做基础，再补 B 的角色线条清晰度和 A 的 UI 可读性。
- 当前更推荐 C 作为气质母版，B 作为角色/敌人精修参考，A 作为背包结构和交互可读性参考。

## 12. A 版主角 Q 版动作试稿

试稿文件：

本轮目标：

- 验证 image2 能否按参考图保持角色一致性。
- 明确九宫格读取顺序，避免动作帧乱序。

九宫格提示词规则：
- `3 columns x 3 rows`。
- 按照从左到右、从上到下读取。
- 第 1 帧为左上，第 9 帧为右下。
- 提示词必须逐帧列出动作，要求动作从第 1 帧到第 9 帧按时间顺序推进。

视觉检查结论：
- 主角标准图成熟度比旧 Q 版素材更好，适合版本 A。
- 走路九宫格顺序基本可读，角色一致性较好，动作更像持剑行走/慢跑。
- 挥剑九宫格动作顺序明显改善，但第 3 帧和第 7 帧存在较大身体转向；如果直接接入，动作会有更强扭身感。

下一步接入建议：
- 先让用户确认挥剑表是否接受“扭身挥剑”。
- 如果接受，则本地抠绿、裁切、替换 `assets/chibi/chibi_hero_walk_sheet.png` 和 `assets/chibi/chibi_hero_attack_sheet.png`。
- 如果不接受，则重做一张“全程面向右、无背向转身”的保守挥剑表，再接入。

## 7. 后续生成提示词基线

通用正向：

```text
Bright colorful Japanese fairy-tale RPG, polished commercial 2D game art, soft storybook watercolor with clean anime character design, warm green, cream, sky blue, sakura pink, coral accents, lightweight cloth and painted wood UI, readable at 1280x720, cohesive background characters and UI, no readable text.
```

通用反向：

```text
Avoid dark mood, greasy rendering, western fantasy, heavy metal, armor, gothic horror, muddy brown palette, over-detailed clutter, AI blur, unreadable tiny details, thick heavy UI, mismatched style, floating characters.
```

生成前检查：
- 是否能一眼看出这是同一个游戏。
- 是否有清晰地面和角色落脚点。
- 背包是否是核心操作区，而不是普通装饰。
- 是否没有文字、logo、水印和多余符号。
- 是否符合 `docs/source/` 的剧本世界观。
