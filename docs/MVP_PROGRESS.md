# 《记忆背包》MVP 开发规划与完成度

更新时间：2026-06-01

本文档用于跟踪 MVP-1 开发进度。每完成一轮开发，都要更新本文件中的模块状态、总完成度、已完成内容、下一步任务，然后提交并推送。

## 1. 范围边界

当前只做 MVP-1，严格以 `docs/source/` 中的设计文档为准。

MVP-1 包含：

- 序章：故乡。
- 第一章：遗忘森林，1000m。
- 自动行进。
- 自动战斗。
- 4 格记忆背包。
- 记忆获得、放弃、丢弃、替换。
- 核心记忆二次确认。
- JSONL 脚本播放。
- `choice.options[].effects` 执行。
- 3 场普通战斗。
- 1 场 Boss：忘名猎人。
- 最小世界反馈：名字栏、关系回应、结尾分支。
- 调试面板。

MVP-1 暂不做：

- 记忆合成。
- 第二章、第三章、终章。
- 完整五结局。
- 离线收益。
- 复杂商店。
- 成就。
- 装备系统。
- 技能树。
- 正式音频系统。
- 正式美术资源管线。

## 5. 近期开发顺序

### 下一轮：M2 记忆背包第一版

目标：

- 把当前“背包摘要文本”升级为 4 格记忆卡 UI。
- 每张卡显示名称、标签、数值效果、关系对象、承诺摘要。
- 为替换界面打基础。

验收：

- 标准开局后 4 张初始记忆以卡片形式显示。
- 轻装开局后显示 3 张记忆。
- 玩家能直接看到每张记忆连接着谁。

### 再下一轮：M2 满包替换

目标：

- `F0010` 选择新记忆时，如果背包满，打开替换界面。
- 支持丢弃旧记忆接受新记忆。
- 支持放弃新记忆。
- 核心记忆二次确认。

验收：

- 标准开局到 `F0010` 拾取“有人等我”时，必须处理满包替换。
- 丢弃“母亲做的热汤”后进入 discarded。
- 丢弃“我的名字”后名字栏变成“勇者”。

### 第三轮：M3 自动行进

目标：

- 用 `chapter_flow.json` 驱动森林 0m 到 1000m。
- 按 progress 触发事件和战斗占位。
- 事件播放时暂停行进。

验收：

- 序章结束后进入森林。
- 森林自动推进到 `F0010/F0021/F0034/F0040` 等关键节点。

## 6. 图片生成计划

当前图片生成消耗：0 / 50 张（2026-06-01）

短期不批量生成图片。等核心循环可以跑通后，先生成美术规划中建议的 6 张风格测试图：

1. `bg_village_dawn`
2. `bg_forest_path`
3. `hero_default`
4. `mother_warm`
5. `icon_memory_mothers_soup`
6. `fx_slash_basic_sheet`

生成规则：

- 每生成一张，必须写入 `logs/image_generation.jsonl`。
- 不记录密钥、账号密码、token 或临时敏感 URL。
- 单日最多 50 张。
- 未确认风格前，不进入 36 张第一批美术批量生成。

## 7. 每轮更新规则

每轮开发完成后必须做：

1. 回顾 `docs/source/` 相关文档，确认没有偏离。
2. 更新本文档：
   - 当前总完成度。
   - 模块状态。
   - 已完成内容。
   - 下一步计划。
   - 图片生成消耗。
3. 运行对应验证。
4. 检查敏感信息。
5. 提交并推送。

## 8. 当前风险

- 脚本播放器已能跑序章，但森林脚本还未完整接入手动流程。
- 背包现在只是摘要文本，还没有满足“卡片必须显示关系对象和承诺”的 UI 要求。
- 满包替换和核心记忆二次确认尚未实现，这是 MVP 核心风险。
- 自动战斗和自动行进尚未接入，当前还不是可玩循环。
- 目前没有真实美术资源，后续要先小批量确认风格，不能直接生成 36 张。
