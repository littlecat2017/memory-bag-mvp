# 《记忆背包》MVP 开发规划与完成度

更新时间：2026-06-01

本文档用于跟踪 MVP-1 开发进度。每完成一轮开发，都要更新本文件中的模块状态、总完成度、已完成内容、下一步任务，然后提交并推送。

## 1. 范围边界

当前只做 MVP-1，严格以 `docs/source/` 中的设计文档为准。

MVP-1 包含：

- 序章：故乡。
- 第一章：遗忘森林，1000m。
- 自动行进。
- 自动战斗。
- 4 格记忆背包。
- 记忆获得、放弃、丢弃、替换。
- 核心记忆二次确认。
- JSONL 脚本播放。
- `choice.options[].effects` 执行。
- 3 场普通战斗。
- 1 场 Boss：忘名猎人。
- 最小世界反馈：名字栏、关系回应、结尾分支。
- 调试面板。

MVP-1 暂不做：

- 记忆合成。
- 第二章、第三章、终章。
- 完整五结局。
- 离线收益。
- 复杂商店。
- 成就。
- 装备系统。
- 技能树。
- 正式音频系统。
- 正式美术资源管线。

## 7. 每轮更新规则

每轮开发完成后必须做：

1. 回顾 `docs/source/` 相关文档，确认没有偏离。
2. 更新本文档：
   - 当前总完成度。
   - 模块状态。
   - 已完成内容。
   - 下一步计划。
   - 图片生成消耗。
3. 运行对应验证。
4. 检查敏感信息。
5. 提交并推送。
