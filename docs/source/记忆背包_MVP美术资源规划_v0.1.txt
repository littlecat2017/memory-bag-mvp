一、总体原则

MVP 美术目标不是做完正式版，而是验证三件事：

1. 玩家能立刻看懂这是一个王道勇者旅途。
2. 玩家能从画面变化中感到“关系被保留或切断”。
3. 程序可以用这些资源快速搭出可玩的原型。

MVP 不追求：
1. 全角色完整动作。
2. 全章节背景。
3. 高成本逐帧角色动画。
4. 复杂 UI 皮肤。
5. 正式 Steam 商店级宣传图。

MVP 追求：
1. 背景氛围统一。
2. 立绘能支撑剧情阅读。
3. 记忆卡图标能表达关系对象。
4. 战斗特效足够清楚。
5. 所有资源规格固定，便于程序直接接入。


二、推荐美术方向

整体风格：
低饱和日式奇幻 + 记忆褪色质感。

表现方式：
2D 插画背景 + 半身立绘 + 简化 Q 版战斗单位 + 九宫格特效 spritesheet。

不要做成过度黑暗、恐怖或纯抽象风格。
第一眼仍然应该像传统勇者 RPG：
村庄、森林、营火、猎人、魔物、旅途。

主题表达靠细节：
1. 保留关系时，画面里有可辨认的物件、光源、名字、影子。
2. 丢弃关系时，物件还在，但它失去指向某个人的细节。
3. 名字相关画面要避免直接煽情，重点表现“可被呼唤”和“无法落名”。

四、统一规格

1. 背景图

规格：
1920 x 1080

格式：
PNG

用途：
剧情背景、场景氛围、章节节点。

要求：
画面中心偏上留出阅读空间。
下方 35% 不要放太多关键细节，因为可能被对话框遮挡。
左右边缘可以做延展，方便宽屏裁切。

2. 角色立绘

规格：
1024 x 1536

格式：
PNG，透明背景。

用途：
剧情对话。

构图：
半身或膝上。
人物面向屏幕 3/4 角度。
头部和上半身细节清晰。

3. 战斗单位图

规格：
512 x 512

格式：
PNG，透明背景。

用途：
自动战斗中的勇者、敌人、Boss。

要求：
统一视角。
可做轻微上下浮动、缩放、受击闪白，不要求逐帧动作。

4. 记忆卡图标

规格：
512 x 512

格式：
PNG，透明背景或简单底色。

用途：
记忆背包、替换界面、获得记忆弹窗。

要求：
每张图标必须一眼能对应关系对象或承诺，不要只做抽象符号。

5. UI/状态图标

规格：
256 x 256

格式：
PNG，透明背景。

用途：
名字栏、背包容量、核心记忆、关系反馈、战斗日志。

6. 战斗特效九宫格

规格：
1536 x 1536

单帧：
512 x 512

布局：
3 x 3，共 9 帧。

格式：
PNG，透明背景。

用途：
普通攻击、受击、恢复、记忆触发、无名侵蚀、Boss 技能等。

播放建议：
12 fps。
9 帧总时长约 0.75 秒。
程序可按 0-8 帧顺序播放，也可抽取其中 4-6 帧做短效果。

角色立绘模板：
低饱和日式奇幻 RPG 半身角色立绘，透明背景，1024x1536，3/4 视角，清晰面部表情，服装简洁可识别，适合视觉小说对话，不要背景，不要文字，不要 UI。

记忆图标模板：
低饱和日式奇幻物件图标，512x512，中心构图，清晰剪影，轻微旧纸和记忆光点质感，适合游戏道具图标，不要文字，不要 UI。

战斗单位模板：
低饱和日式奇幻 RPG 战斗单位，512x512，透明背景，小比例全身，清晰轮廓，适合自动战斗界面，朝向统一，不要背景，不要文字。

九宫格特效模板：
透明背景 spritesheet，1536x1536，3x3 grid，9 frames，each frame 512x512，game VFX animation frames，clear progression from frame 1 to frame 9，不要角色，不要文字，不要背景。


十五、验收标准

每张图生成后检查：
1. 是否符合规格。
2. 是否没有文字、水印、UI。
3. 是否能在小尺寸下看清主体。
4. 是否和“低饱和日式奇幻 + 记忆褪色”方向一致。
5. 是否服务“关系凭证”主题。

九宫格特效额外检查：
1. 是否真的是 3 x 3。
2. 每帧主体是否在 512 x 512 格内。
3. 帧之间是否有明确动画变化。
4. 是否透明背景。
5. 播放时是否能在 0.75 秒内读懂效果。
