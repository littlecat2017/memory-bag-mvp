一、项目定位

游戏名：
《记忆背包》

类型：
放置行进 + 自动战斗 + 记忆背包管理 + 短篇分支叙事。

目标平台：
PC / Steam 优先。

MVP 时长：
10-15 分钟。

正式版时长：
40-90 分钟。

一句话概念：
勇者会自动向魔王城前进，玩家真正要做的不是操作战斗，而是在有限背包中选择保留、丢弃或压缩哪些记忆；每段记忆都连接着一个人、一句承诺或一种世界对勇者的回应。

当前主题原则：
记忆不是单纯的自我碎片，也不是为了制造悲伤而存在的文本。
记忆是关系凭证。
丢掉记忆，不只是少一张卡，而是切断一段关系、一句承诺、一个名字被回应的方式。


二、核心体验

玩家大部分时间看勇者自动前进、打怪、升级。

关键操作只发生在少数节点：
1. 选择初始记忆。
2. 遇到新记忆时决定是否放进背包。
3. 背包满时决定丢掉哪段旧记忆。
4. 在 Boss 或关卡门槛前决定是否交出特定记忆。
5. 在合成系统开放后，决定是否把具体回忆压缩成更强的抽象能力。

本作希望制造的停顿不是：
“我丢掉它会不会很难过？”

而是：
“我丢掉它之后，谁还会等我？”
“我得到力量以后，是不是把某个具体的人压成了工具？”
“我继续前进，是不是让世界少了一种回应我的方式？”

成功标准：
玩家不只记得某张记忆的数值，还能说出它连接的人、承诺和世界反馈。


三、世界观

表层世界观：
王道日式奇幻。

包含：
故乡、森林、山道、无名城、魔王城、勇者、魔物、营火、Boss、旅途。

真实世界观：
世界正在被“忘却”侵蚀。

这里的忘却不是单纯记忆消失，而是关系失去落点：
名字无人呼唤。
约定无人履行。
等待无人回应。
记录仍在，但记录找不到那个具体的人。

魔物：
魔物不是普通怪物，而是被关系断裂、名字空洞化、承诺失效后留下的残影。

魔王：
魔王不是单纯外部邪恶，也不只是主角缺失的一部分。
它是没人再回应的名字、约定、等待和关系的集合。

勇者的旅途：
勇者为了继续前进，必须不断管理记忆。
有些记忆能让他变强。
有些记忆几乎没有战斗价值，却让某个人、某个地方、某句承诺还能和他发生关系。

核心矛盾：
为了抵达终点，我必须不断削减背包。
但我削减的不是负重，而是世界和我之间的连接方式。


四、主角与关键角色

主角：艾尔

身份：
前往魔王城的勇者。

玩家一开始不需要知道他完整过去，只需要知道他正在前进。
他的过去通过记忆卡逐渐显露。

母亲：
对应记忆“母亲做的热汤”。
她不是纯煽情角色，而是“有人希望你活着回来”的关系凭证。

师父：
对应记忆“第一次拿起木剑”“师父的训斥”。
他代表训练、教导和“有人相信你能保护别人”。

莉娅：
对应记忆“出发的理由”。
她不是任务发布器，而是把某件事托付给勇者的人。

魔王：
终点处的关系集合。
它会根据玩家一路保留和丢弃的记忆，改变质问方式和结局方向。


五、核心玩法循环

基础循环：
1. 勇者自动前进。
2. 自动遭遇敌人并战斗。
3. 获得经验、金币、记忆碎片或新记忆。
4. 到达记忆节点。
5. 玩家决定是否接纳新记忆。
6. 背包已满时，玩家必须保留、丢弃或替换记忆。
7. 记忆改变数值、战斗日志、NPC 对话、场景状态和结局条件。
8. 到达章节 Boss。
9. Boss 根据玩家保留/丢弃的关系凭证改变行为或台词。
10. 章节结尾根据关键记忆状态给出反馈。

玩家操作频率：
低频操作。
关键选择高权重。

离线收益原则：
可以给经验、金币、普通素材。
不能自动替玩家获得、丢弃或合成核心记忆。


六、记忆背包系统

背包容量：
MVP：4 格。
正式版：可扩展到 7 格。
部分沉重记忆可占 2 格。

记忆字段：
id
name
tags
slot_cost
stats
relation_target
relation_type
obligation
owned_world_response
discard_world_response
ui_loss_hint

设计原则：
每张重要记忆都必须回答四个问题：
1. 它连接着谁？
2. 它代表什么承诺、等待或关系？
3. 保留它时，世界如何回应？
4. 丢掉它时，世界哪里发生了变化？

核心记忆：
核心记忆可以丢弃，但必须二次确认。
核心记忆不应简单加攻击力。
它们的价值主要体现在世界反馈、Boss 选择和结局条件。


七、记忆示例

母亲做的热汤
关系对象：母亲。
关系类型：亲情。
代表承诺：有人希望你活着回来。
数值效果：营火恢复量提升。
保留反馈：营火、空碗和故乡 NPC 会把勇者重新指向家。
丢弃反馈：空碗仍在，但它只是一只空碗，不再通向母亲。

第一次拿起木剑
关系对象：师父。
关系类型：师承。
代表承诺：有人曾经相信你能学会保护别人。
数值效果：普通攻击提升。
保留反馈：师父回响会把战斗动作还原成一次具体教导。
丢弃反馈：勇者仍会挥剑，但剑法不再指向教他的人。

出发的理由
关系对象：莉娅。
关系类型：托付。
代表承诺：替一个快要忘记一切的人继续记得。
数值效果：记忆抵抗提高。
保留反馈：终章能回答“是谁把你送到这里”。
丢弃反馈：任务目标还在，但托付它的人从旅途中消失。

我的名字
关系对象：世界。
关系类型：称呼。
代表承诺：让别人的等待、呼唤和记录能落到一个具体的人身上。
数值效果：低直接收益。
保留反馈：NPC、结尾和 UI 还能以名字回应勇者。
丢弃反馈：名字栏被擦除，世界还能记录勇者行为，却越来越难回应他本人。

空名牌
关系对象：失去名字的人。
关系类型：代偿。
代表承诺：替一个喊不出名字的人带走空白。
数值效果：对无名敌人伤害提高。
代价：自身名字相关反馈被削弱或关闭。


八、战斗系统

战斗形式：
自动战斗。

玩家不直接操作攻击、防御和技能释放。
战斗表现由记忆组合决定。

MVP 战斗目标：
战斗要提供压力，但不能抢走主题。
敌人和 Boss 需要作为记忆选择的测试器。

战斗反馈方式：
1. 数值变化。
2. 战斗日志中触发记忆文本。
3. 敌人对特定记忆状态产生反应。
4. Boss 前后改变台词和结尾。

失败原则：
原型阶段避免高惩罚。
失败可以掉进度、损失普通资源或触发警告。
不要随机删除核心记忆。


九、合成系统

合成定位：
合成不是单纯升级。
合成是把具体关系压缩成更有效率的能力。

合成收益：
1. 提升数值。
2. 节省背包空间。
3. 解锁流派。

合成代价：
1. 具体人物淡出。
2. 对话选项减少。
3. 部分世界反馈关闭。
4. 结局条件改变。

示例：
第一次拿起木剑 + 师父的训斥 = 战斗本能。
收益：攻击和暴击提升。
代价：师父从具体教导者变成战斗动作的来源，相关回响减少。

母亲做的热汤 + 雨夜的灯 = 想回家。
收益：营火恢复和低血保护增强。
代价：前进速度下降，勇者更频繁回头。


十、章节结构

序章：故乡
目标：让玩家获得初始记忆，并理解背包容量有限。
主题：出发不是抽象使命，而是被具体的人送上路。

第一章：遗忘森林
目标：验证 MVP 核心循环。
主题：第一批取舍让玩家看见“关系反馈”。
Boss：忘名猎人。
关键选择：是否献出名字。

第二章：灰烬山道
目标：开放合成系统。
主题：具体关系被压缩成能力。
Boss：灰骑士。

第三章：无名城
目标：展示制度化遗忘。
主题：记录可以保存条目，但保存不了呼唤、气味和等待。
Boss：无名王。

终章：魔王城
目标：回收玩家所有关键取舍。
主题：魔王是无人回应的关系集合。


十一、结局方向

结局不按道德好坏简单分类，而按“勇者如何回应关系”分类。

可能方向：
1. 回家结局：保留较多故乡和等待关系。
2. 斩断结局：为了胜利切断多数关系。
3. 无名结局：名字被交出，世界记录行为但无法回应本人。
4. 托付结局：保留出发理由，能回应莉娅的托付。
5. 和解结局：承认被切断的名字、约定和等待仍然需要被回应。

结局判断重点：
不是玩家是否“完整”。
而是玩家最终还愿意回应哪些人，以及哪些关系已经无法被回应。


十二、美术风格

整体方向：
传统日式奇幻的可读性 + 低饱和记忆质感。

画面目标：
第一眼像熟悉的勇者旅途。
玩下去逐渐发现世界不是被黑暗吞噬，而是被关系空洞化。

推荐风格：
2D 像素 / 伪 16-bit / 轻 HD-2D 均可。
MVP 阶段使用简洁占位图即可，但 UI 信息层级必须清楚。

颜色原则：
故乡：温暖但不过度怀旧。
森林：青绿、灰雾、留白。
山道：灰、红、炭黑，强调压缩与燃烧。
无名城：白、灰、规整线条，强调制度和编号。
魔王城：低饱和黑白，少量代表记忆的颜色残留。

记忆卡 UI：
每张卡必须同时展示：
1. 名称。
2. 数值效果。
3. 关系对象。
4. 丢弃提示。

核心记忆视觉：
不靠大红警告吓玩家。
用名字栏、人物剪影、场景物件淡化来表现关系被切断。


十三、声音风格

整体原则：
少量、克制、重复可听。

自动行进：
轻脚步、环境底噪。

记忆获得：
短促清亮音。

记忆丢弃：
低频抽离感，像环境声少了一层。

名字相关选择：
短暂停顿或静音。

Boss：
音乐不只强化战斗，也要让玩家意识到某些声音来自曾经被切断的关系。


十四、MVP 验证目标

MVP 不验证完整剧情。
MVP 只验证三件事：

1. 自动行进和自动战斗是否能承载低频选择。
2. 背包满时的记忆取舍是否有足够压力。
3. 玩家是否能理解记忆是关系凭证，而不只是装备或文本收藏。

MVP 通过标准：
玩家在至少一个记忆选择上停顿。
玩家能说出至少一张记忆连接着谁。
玩家注意到丢弃记忆后世界反馈发生变化。
玩家愿意再玩一次尝试不同取舍。
